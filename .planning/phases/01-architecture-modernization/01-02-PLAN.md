**Step 1: Create domain files by splitting convex/app.ts**

- `convex/billing/actions.ts` -- Stripe checkout, portal actions
- `convex/billing/mutations.ts` -- subscription-related mutations (if any in stripe.ts)

**Step 2: Add Zod validation to mutations**

For mutations that accept user input, use `zCustomMutation` from `convex-helpers/server/zod4`:

```typescript
import { zCustomMutation } from "convex-helpers/server/zod4";
import { NoOp } from "convex-helpers/server/customFunctions";
import { mutation } from "@cvx/_generated/server";
```

In Convex files, import Zod as `import { z } from "zod/v4"` (NOT `from "zod"`).
Import shared schemas: `import { username } from "~/src/shared/schemas/username"` (use the `~/` alias which maps to project root).

Apply zCustomMutation to:
- `updateUsername` -- validate with username schema
- `completeOnboarding` -- validate username

For billing enums, use `zodToConvex()` from `convex-helpers/server/zod4` to derive Convex validators from the shared Zod schemas where applicable.

**Step 3: Update ALL api.* import paths**

After moving files, Convex regenerates `_generated/api.d.ts`. The old paths like `api.app.getCurrentUser` become `api.users.queries.getCurrentUser`.

Search the ENTIRE codebase for `api.app.` and `api.stripe.` and `internal.stripe.` and update every reference:
- Frontend files (useQuery, useMutation calls)
- Test files (convex/app.test.ts -> split into convex/users/*.test.ts etc.)
- Other Convex files that reference internal functions

Common mapping:
- `api.app.getCurrentUser` -> `api.users.queries.getCurrentUser`
- `api.app.updateUsername` -> `api.users.mutations.updateUsername`
- `api.app.updateUserImage` -> `api.users.mutations.updateUserImage`
- `api.app.removeUserImage` -> `api.users.mutations.removeUserImage`
- `api.app.completeOnboarding` -> `api.onboarding.mutations.completeOnboarding`
- `api.app.generateUploadUrl` -> `api.uploads.mutations.generateUploadUrl`
- `api.app.getActivePlans` -> `api.billing.queries.getActivePlans`
- `api.stripe.*` -> `api.billing.stripe.*` or `api.billing.actions.*`
- `internal.stripe.*` -> `internal.billing.stripe.*`
