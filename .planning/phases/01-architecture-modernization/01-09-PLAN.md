<task type="auto">
  <name>Task 2: Wire zCustomMutation into user and onboarding mutations</name>
  <files>convex/users/mutations.ts, convex/onboarding/mutations.ts</files>
  <action>
Replace standard `mutation()` with `zCustomMutation` for mutations that accept user input, so the shared Zod schemas validate on the server.

**convex/users/mutations.ts:**

1. Add imports:
   ```typescript
   import { zCustomMutation } from "convex-helpers/server/zod4";
   import { NoOp } from "convex-helpers/server/customFunctions";
   import { username as usernameSchema } from "~/src/shared/schemas/username";
   ```

2. Create the zMutation wrapper:
   ```typescript
   const zMutation = zCustomMutation(mutation, NoOp);
   ```

3. Convert `updateUsername` to use zMutation:
   ```typescript
   export const updateUsername = zMutation({
     args: { username: usernameSchema },
     handler: async (ctx, args) => {
       const userId = await auth.getUserId(ctx);
       if (!userId) return;
       await ctx.db.patch(userId, { username: args.username });
     },
   });
   ```

4. Leave `updateUserImage`, `removeUserImage`, and `deleteCurrentUserAccount` as regular `mutation()` -- they don't accept user-typed input that benefits from Zod validation (imageId is a storage ID, the others take no args). Only `updateUsername` has a user string that should be validated.

5. Remove the `v` import from `convex/values` only if no longer used. `updateUserImage` still uses `v.id("_storage")`, so keep it.

**convex/onboarding/mutations.ts:**

1. Add imports:
   ```typescript
   import { zCustomMutation } from "convex-helpers/server/zod4";
   import { NoOp } from "convex-helpers/server/customFunctions";
   import { username as usernameSchema } from "~/src/shared/schemas/username";
   import { currency as currencySchema } from "~/src/shared/schemas/billing";
   ```

2. Create the zMutation wrapper:
   ```typescript
   const zMutation = zCustomMutation(mutation, NoOp);
   ```

3. Convert `completeOnboarding` to use zMutation:
   ```typescript
   export const completeOnboarding = zMutation({
     args: { username: usernameSchema, currency: currencySchema },
     handler: async (ctx, args) => {
       // ... same handler logic
     },
   });
   ```

4. Remove the `currencyValidator` import from `@cvx/schema` and the `v` import from `convex/values` (no longer needed since both args are now Zod schemas).

**CRITICAL:** The `zCustomMutation` changes the API contract. The Convex type system will generate new types in `_generated/api.d.ts`. Existing tests call these mutations with the same argument shapes (`{ username: "testuser" }`, `{ currency: "usd" }`), so tests should continue to pass. But verify by running the full test suite.

**CRITICAL:** `zCustomMutation` with `NoOp` does NOT add any middleware -- it only adds Zod validation. The `NoOp` import from `convex-helpers/server/customFunctions` is the identity modifier. This is the documented pattern from convex-helpers.
