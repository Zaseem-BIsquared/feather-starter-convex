**CRITICAL IMPORT NOTE:** In Convex files, Zod must be imported as `from "zod/v4"` (not `from "zod"`) because convex-helpers/server/zod4 internally uses zod/v4/core types. However, the shared schemas import `from "zod"` which resolves to v4 in this project. The zodToConvex function accepts standard Zod types, so importing the schemas directly should work.

**Changes to convex/schema.ts:**

1. Add imports at top:
   ```typescript
   import { zodToConvex } from "convex-helpers/server/zod4";
   import { currency as currencySchema, interval as intervalSchema, planKey as planKeySchema, CURRENCY_VALUES, INTERVAL_VALUES, PLAN_KEY_VALUES } from "../src/shared/schemas/billing";
   ```

2. Replace the manually-defined validators with zodToConvex-derived ones:
   ```typescript
   // BEFORE:
   export const CURRENCIES = { USD: "usd", EUR: "eur" } as const;
   export const currencyValidator = v.union(v.literal(CURRENCIES.USD), v.literal(CURRENCIES.EUR));

   // AFTER:
   export const CURRENCIES = { USD: "usd", EUR: "eur" } as const;
   export const currencyValidator = zodToConvex(currencySchema);
   ```

   Do the same for intervalValidator and planKeyValidator.

3. Keep the CURRENCIES, INTERVALS, PLANS const objects as they are -- they're used for property access (e.g., `CURRENCIES.USD`, `INTERVALS.MONTH`). But now the Zod schemas are the source of truth for validation, not the manually-defined v.union/v.literal chains.

Run `npm run typecheck` after changes to verify Convex type inference still works correctly with zodToConvex-derived validators. The zodToConvex return type should be compatible with everywhere currencyValidator, intervalValidator, and planKeyValidator are used (defineTable fields, function args).


3. Convert `completeOnboarding` from `mutation()` to `zMutation()`:
   ```typescript
   export const completeOnboarding = zMutation({
     args: { username, currency: currencyZod },
     handler: async (ctx, args) => {
       const userId = await auth.getUserId(ctx);
       if (!userId) return;
       const user = await ctx.db.get(userId);
       if (!user) return;
       await ctx.db.patch(userId, { username: args.username });
       if (user.customerId) return;
       await ctx.scheduler.runAfter(
         0,
         internal.billing.stripe.PREAUTH_createStripeCustomer,
         { currency: args.currency, userId },
       );
     },
   });
   ```
