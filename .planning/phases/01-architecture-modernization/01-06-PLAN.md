**Step 1: Install Plop.js**

```bash
npm install --save-dev plop
```

**Step 2: Add npm scripts to package.json**

Add to `scripts`:
```json
"gen:feature": "plop feature",
"gen:route": "plop route",
"gen:convex-function": "plop convex-function",
"gen:form": "plop form"
```

**Step 3: Create plopfile.js**

Create `plopfile.js` (ESM export) with 4 generators. Follow the pattern from RESEARCH.md but with full wired-up templates (not minimal stubs, per user decision).

**Generator 1: feature**
Prompts: name (string)
Creates:
- `src/features/{{name}}/components/{{pascalCase name}}Page.tsx` -- sample page component with useTranslation hook
- `src/features/{{name}}/hooks/use{{pascalCase name}}.ts` -- sample custom hook
- `src/features/{{name}}/index.ts` -- barrel export
- `src/features/{{name}}/{{name}}.test.tsx` -- test file with renderWithRouter setup
- `src/features/{{name}}/README.md` -- per-feature README from template
- `convex/{{name}}/queries.ts` -- sample query with auth check
- `convex/{{name}}/mutations.ts` -- sample mutation with zCustomMutation and Zod validation

**Generator 2: route**
Prompts: name (string), authRequired (confirm)
Creates:
- Route file at `src/routes/_app/_auth/{{name}}.tsx` (if authRequired) or `src/routes/_app/{{name}}.tsx` (if not)
- Thin wrapper pattern: imports component from feature, creates route with beforeLoad

**Generator 3: convex-function**
Prompts: domain (string), type (list: query/mutation/action), name (string)
Creates:
- `convex/{{domain}}/{{pluralize type}}.ts` or appends to existing file
- Includes auth check boilerplate, proper imports from `@cvx/_generated/server`
- If mutation, includes zCustomMutation pattern with placeholder Zod schema

**Generator 4: form**
Prompts: name (string), feature (string)
Creates:
- `src/shared/schemas/{{name}}.ts` -- Zod schema with placeholder fields
- `src/features/{{feature}}/components/{{pascalCase name}}Form.tsx` -- TanStack Form component using the schema, with shadcn/ui Input fields

**Step 4: Create all Handlebars template files**

Create `templates/` directory with subdirectories for each generator. Each `.hbs` file should produce a working, importable file -- not a stub. Include:
- Proper TypeScript types
- Import statements that resolve
- Sample content that demonstrates the pattern
- Comments explaining customization points

**Step 5: Test generators**

Run each generator with test inputs to verify output:
```bash
npx plop feature -- --name test-feature
# Verify files created, then delete them
rm -rf src/features/test-feature convex/test-feature
```
