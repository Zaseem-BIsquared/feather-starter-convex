**Plugin shell script (scripts/plugin.sh):**

Create the script with three commands:
- `list` -- lists all remote `plugin/*` branches
- `preview <branch>` -- shows diff stat of branch vs main
- `install <branch>` -- merges the plugin branch into current branch with `--no-edit`

Make it executable (`chmod +x`). Include usage help and error handling (missing branch arg, branch not found). Use the pattern from RESEARCH.md.

**Auto-rebase workflow (.github/workflows/rebase-plugins.yml):**

Triggers on push to `main`. For each `plugin/*` remote branch:
1. Checks it out
2. Attempts `git rebase main`
3. If rebase succeeds, force-pushes with `--force-with-lease`
4. If rebase fails, aborts and creates a GitHub issue with conflict details

Uses `actions/checkout@v4` with `fetch-depth: 0` and `token: ${{ secrets.GITHUB_TOKEN }}`.

**Plugin CI workflow (.github/workflows/plugin-ci.yml):**

Triggers on push to `plugin/**` branches. Runs:
1. `npm ci`
2. `npm run typecheck`
3. `npm run lint`
4. `npm run test`

Uses `actions/setup-node@v4` with node 20 and npm cache.

Both workflow files follow the exact patterns from RESEARCH.md code examples.
